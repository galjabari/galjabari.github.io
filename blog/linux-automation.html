Linux Automation with Ansible

To automate tasks on Linux systems such as installing software, configuring services, and managing users, you can use Ansible automation tool.

Install Ansible on Ubuntu

To install Ansible on your Linux machine (control node), run the following commands:

sudo apt update
sudo apt install software-properties-common -y
sudo add-apt-repository --yes --update ppa:ansible/ansible
sudo apt install ansible -y

Verify the installation by checking the Ansible version:

ansible --version
Create an Ansible inventory file

To manage and configure Linux systems with Ansible, you need to create a file that lists Linux hosts with their settings:

nano hosts.ini

For example, add the following lines to define a group of servers:

[servers]
server1 ansible_host=192.168.1.101
server2 ansible_host=192.168.1.102
[servers:vars]
ansible_user=ubuntu
ansible_password=ubuntu
ansible_host_key_checking=false

Ensure that the SSH service is installed and properly configured on the target servers to enable remote management.

Write an Ansible playbook

To verify connectivity with the target servers and ensure that Ansible can execute tasks on the remote systems, you can use the ping module within an Ansible playbook:

nano ping.yml

Add the following lines in the playbook:

---
- hosts: servers
  gather_facts: false
  tasks:
    - name: Check connectivity
      ansible.builtin.ping:

Run the playbook using the ansible-playbook command:

ansible-playbook -i hosts.ini ping.yml

This will ping all hosts defined in the playbook and return a "pong" response for each reachable host.

To define system administration tasks using Ansible, you should create a playbook. For example:

nano install_apache.yml

Add the following lines to install Apache on Ubuntu servers:

---
- name: Install Apache on Ubuntu servers
  hosts: servers
  become: yes
  tasks:
    - name: Update repositories cache and install apache2
      ansible.builtin.apt:
        name: apache2
        state: latest
        update_cache: yes

    - name: Start and enable apache2 service
      ansible.builtin.service:
        name: apache2
        state: started
        enabled: yes

Run the playbook:

ansible-playbook -i hosts.ini install_apache.yml

Open a web browser and enter the IP addresses of server1 and server2 in the address bar; you should see the default Apache2 Ubuntu page.
