Install and configure Mail Server on Ubuntu

Before setting up a mail server on Ubuntu, you need to complete the following guide:

Install and configure an authoritative DNS Server on Ubuntu

Next, edit the DNS zone file.

sudo nano /etc/bind/db.example.com

Add the following lines under the existing records:

@       IN      MX 10   mail.example.com.
mail    IN      A       192.168.1.10
smtp    IN      CNAME   mail.example.com.
imap    IN      CNAME   mail.example.com.

Replace mail.example.com with your mail server's hostname.

After making changes, restart the BIND service to apply the configuration.

sudo systemctl restart bind9.service

Verify that BIND is running without errors.

sudo systemctl status bind9.service

Finally, test DNS resolution for your domain:

dig MX example.com

This test will list MX records for a domain in priority order.

Install and configure SMTP Server

First, set the hostname for your mail server.

sudo hostname mail.example.com

Update package lists and install Postfix, which is the SMTP server.

sudo apt update
sudo apt install postfix -y

After installation, you will be prompted to configure Postfix. Accept default configuration by pressing Enter for each question.

To reconfigure Postfix, run the following command:

sudo dpkg-reconfigure postfix

Step 1: Select Internet Site.

Step 2: Enter example.com.

Step 3: Enter admin.

Step 4: Enter example.com, localhost

Step 5: Select No.

Step 6: Enter 127.0.0.0/8 [::1]/128

Step 7: Enter 0

Step 8: Enter +

Step 9: Enter all

Set the mailbox format to Maildir using the following command:

sudo postconf -e 'home_mailbox = Maildir/'
