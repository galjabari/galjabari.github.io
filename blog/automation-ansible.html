Network Automation with Ansible

To automate network tasks across a wide range of network devices and platforms, you can use Ansible tool.

Install Ansible on Ubuntu

To install Ansible on your control node machine, run the following commands:

sudo apt update
sudo apt install software-properties-common -y
sudo add-apt-repository --yes --update ppa:ansible/ansible
sudo apt install ansible -y

Verify the installation by checking the Ansible version:

ansible --version
Create an Ansible inventory file

To manage network devices with Ansible, you need to create a file that lists the network devices. For example:

nano hosts.ini

Add the following lines to define Cisco devices with their IP addresses:

[switches]
switch1 ansible_host=192.168.1.1 ansible_user=admin ansible_password=pass@123
switch2 ansible_host=192.168.1.2 ansible_user=admin ansible_password=pass@123
switch3 ansible_host=192.168.1.3 ansible_user=admin ansible_password=pass@123

Make sure that SSH is configured on these devices for remote management.

Write an Ansible playbook

To automate the configuration of network devices with Ansible, you need to define the configuration tasks using YAML syntax. For example:

nano config_vlans.yml

Add the following lines to define VLAN configuration tasks:

- hosts: switches
  connection: network_cli
  gather_facts: false

  tasks:
    - name: Configure VLANs
      ios_vlan:
        vlan_id: "{{ item.id }}"
        name: "{{ item.name }}"
      loop:
        - { id: 10, name: 'VLAN10' }"
        - { id: 20, name: 'VLAN20' }"

Run the playbook to apply the configuration on Cisco switches:

ansible-playbook config_vlans.yml

Ansible will connect to Cisco switches using SSH, configure the specified VLANs, and report the result.
