# Intrusion Detection with Suricata

First, update package lists to ensure you have the latest versions of all packages:

```bash
sudo apt update
```

You can install Suricata from the official repository:

```bash
sudo apt-get install software-properties-common -y
sudo add-apt-repository --yes ppa:oisf/suricata-stable
sudo apt-get install suricata -y
```

After installation, you need to configure Suricata by editing the main configuration file:

```bash
sudo nano /etc/suricata/suricata.yaml
```

In this file, you need to specify the network interfaces to monitor in the `af-packet` section:

```
af-packet:
    - interface: eth0
```

Replace `eth0` with your actual network interface, which you can find using the `ip addr` command.

Enable and start the Suricata service with the following commands:

```bash
sudo systemctl enable suricata.service
sudo systemctl start suricata.service
```

To update Suricata signatures or rules, you can use the `suricata-update` tool:

```bash
sudo suricata-update
```

After updating the rules, restart the Suricata service to ensure it loads the new rules:

```bash
sudo systemctl restart suricata.service
```

To test the functionality of Suricata, open a new terminal to check the log:

```bash
sudo tail -f /var/log/suricata/fast.log
```

Use `curl` to trigger the rule from the ET Open ruleset, which is written specifically for test.

```bash
curl http://testmynids.org/uid/index.html
```

You should see an alert in the log, including the timestamp and the IP address of your system.

## Custom Suricata Rules for Nmap Scans

To detect Nmap scans, you can create custom Suricata rules. First, create a new file named `local.rules` in the `/var/lib/suricata/rules/` directory:

```bash
sudo nano /var/lib/suricata/rules/local.rules
```

Add the following rules to the `local.rules` file:

```
alert tcp any any -> any any (msg:"NMAP Stealth SYN Scan Detected"; flags:S; threshold:type threshold, track by_src, count 20, seconds 10; sid:1000001; rev:1;)
alert tcp any any -> any any (msg:"NMAP TCP Connect Scan Detected"; flags:S; flow:to_server; threshold:type threshold, track by_src, count 30, seconds 10; sid:1000002; rev:1;)
alert tcp any any -> any any (msg:"NMAP Xmas Scan Detected"; flags:FPU; sid:1000003; rev:1;)
alert tcp any any -> any any (msg:"NMAP FIN Scan Detected"; flags:F; sid:1000004; rev:1;)
alert tcp any any -> any any (msg:"NMAP NULL Scan Detected"; flags:0; sid:1000005; rev:1;)
```

Next, you need to enable these custom rules by adding `local.rules` to the `rule-files` section of your `suricata.yaml` configuration file:

```bash
sudo nano /etc/suricata/suricata.yaml
```

Ensure the `rule-files` section looks like this:

```yaml
rule-files:
  - suricata.rules
  - local.rules
```

Before restarting Suricata, test your configuration to ensure there are no errors:

```bash
sudo suricata -T -c /etc/suricata/suricata.yaml -v
```

If the test is successful, restart the Suricata service to load the new rules:

```bash
sudo systemctl restart suricata
```

To observe the alerts generated by your custom rules, monitor the Suricata log file:

```bash
sudo tail -f /var/log/suricata/fast.log
```

Now, from a separate machine, initiate various Nmap scans against your target IP address. Replace `<TARGET_IP>` with the actual IP address of the machine where Suricata is running.

```bash
sudo nmap -sS <TARGET_IP> # Stealth SYN Scan
sudo nmap -sT <TARGET_IP> # TCP Connect Scan
sudo nmap -sX <TARGET_IP> # Xmas Scan
sudo nmap -sF <TARGET_IP> # FIN Scan
sudo nmap -sN <TARGET_IP> # NULL Scan
```

As these Nmap scans run, you should see corresponding alerts in the Suricata `fast.log`, indicating that your custom rules are successfully detecting the different types of Nmap activity.
